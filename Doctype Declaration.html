Was ist die Doctype Declaration?

  • zusätzlich zu den Tags, Textinhalt und Entities sollte ein HTML-Dokument eine Doctype–Deklaration in der ersten Zeile enthalten
  • der Doctype ist kein HTML–Tag, sondern ein Verweis auf die Document Type Definition (DTD)
  • die DTD ist für den Browser die Anweisung, wie die Seite dargestellt werden soll
  • je nachdem welche DTD verwendet wird, wird ein bestimmter Browser Modus aktiviert, diesen Vorgang nennt man Doctype Sniffing
  • es existieren die Modi standards(=strict), quirks und almost-standard

Die Document Type Declaration im Code

  • <!DOCTYPE html>
<html lang="en">
<head>
<title>A tiny document</title>
</head>
<body>
<h1>Main heading in my document</h1>
<!-- Note that it is "h" + "1", not "h" + the letters "one" -->>
<p>Look Ma, I am coding <abbr title="Hyper Text Markup Language">HTML</abbr>.</p>
</body>
</html>

Standards Mode (=No Quirks Mode)

  • durch die Deklaration <!DOCTYPE html> wird in moderneren Browsern immer der standards mode gewählt
  • der standards mode ermöglicht es, HTML Dokumente fehlerfrei darzustellen

Quirks Mode

  • wenn keine Doctype Declaration erfolgt, tritt der Browser womöglich in den quirks mode (=Kompatibilitätsmodus) ein
  • im quirks-mode simuliert der Browser die Darstellungsweise einer älteren Browser-Version
  • Internet Explorer 9 simuliert im quirks mode zum Beispiel die Umgebung eines IE5

Almost Standard Mode (=Limited Quirks Mode)

  • der almost standard mode ist eine Mischung zwischen standard mode und quirks mode
  • bis auf wenige wichtige Fehlerkorrekturen ist er dem standard mode sehr ähnlich
  • um den almost standard mode aufzurufen ist eine spezielle Doctype Declaration vonnöten:
  • !DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN"
   "http://www.w3.org/TR/html4/loose.dtd">
<head>
<title>An even tinier document</title>
</head>
<body>
<p>Look Ma, I am coding <abbr title="Hyper Text Markup Language">HTML</abbr>.</p>
</body>
</html>
